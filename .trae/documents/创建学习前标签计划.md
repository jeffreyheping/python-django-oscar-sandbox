# 创建"学习前"Git 标签计划

## 目标

为项目当前状态创建一个不可变的快照，方便以后回退。

## 回答你的问题：这样做是最佳实践吗？

**是的，这是非常推荐的版本控制方法！** 用标签标记关键状态是标准做法：

✅ **推荐场景**：
- 项目开始前（学习前）
- 完成重要里程碑后
- 发布正式版本时（v1.0, v2.0）
- 部署到生产环境前
- 任何你想记住的状态

✅ **好处**：
- 语义清晰：一看到标签名就知道这是什么状态
- 易于回退：不用记 commit ID
- 团队共享：标签推送到 GitHub 后团队都能看到
- 比分支更轻量：只是指向 commit 的指针

✅ **常见命名约定**：
- `v1.0.0`, `v2.0.0` - 版本发布
- `dev`, `staging` - 环境标记
- `feature-xxx` - 功能分支
- `学习前`, `基础配置完成` - 个人里程碑

## 背景

这是一个 Django Oscar Sandbox 学习项目，用户希望：
- 在开始深入学习代码之前保存一个干净的起点
- 以后如果改坏代码可以快速恢复到"学习前"状态
- 掌握 Git 版本控制最佳实践

## 操作步骤

### 步骤 1：确认当前状态

```bash
# 查看当前 commit 历史
git log --oneline -5
```

**预期结果**：显示最新的 5 个提交

### 步骤 2：创建"学习前"标签

```bash
# 创建轻量级标签
git tag 学习前
```

**说明**：
- 轻量级标签只是指向特定 commit 的指针
- 不包含额外信息，推荐用于本地标记
- 标签位置固定，不会随着提交移动

### 步骤 3：（可选但推荐）推送标签到远程

```bash
# 推送单个标签
git push origin 学习前

# 或者推送所有标签（谨慎使用）
git push origin --tags
```

**预期结果**：标签出现在 GitHub 远程仓库

### 步骤 4：验证标签创建成功

```bash
# 查看所有标签
git tag -l

# 查看特定标签的详细信息
git show 学习前
```

**预期结果**：显示 "学习前" 标签及其指向的 commit

## 回退方法（未来使用）

如果将来改坏了代码，可以：

```bash
# 方法 1：查看标签信息
git show 学习前

# 方法 2：切换到标签位置（创建临时分支）
git checkout -b 学习前-恢复 学习前

# 方法 3：重置到标签位置（谨慎，会丢失当前工作）
git reset --hard 学习前
```

## 注意事项

⚠️ **重要提示**：
- 标签一旦创建不会移动，始终指向创建时的 commit
- `git checkout 学习前` 会进入"分离头指针"状态，可以安全浏览但不适合直接开发
- 推荐做法：`git checkout -b 新分支 学习前` 创建新分支再修改

## 其他推荐标签命名

| 标签名 | 用途 |
|--------|------|
| `学习前` | 当前（初始状态） |
| `基础完成` | 完成基本配置后 |
| `功能完成` | 实现某个功能后 |
| `v1.0.0` | 正式版本发布 |
